Mantenendo questo stile, trasforma questa pagina in una nuova sezione della piattaforma chiamata Documentazione.

La UI deve restare perfettamente coerente con il design attuale della piattaforma:

• stessa sidebar sinistra
• stesso linguaggio visivo
• stesse proporzioni
• stessi bordi arrotondati
• stessa palette
• stessa pulizia
• stessa sensazione premium/minimal
• niente stile generico da docs template esterno

Obiettivo della pagina

Questa pagina deve rappresentare la sezione Documentazione, separata da Addestramento.

La funzione della sezione è:

• permettere agli utenti di consultare contenuti su violenza online, sicurezza digitale e difesa
• permettere ad admin e operatori di navigare una knowledge base strutturata
• predisporre la pagina a una futura modalità editing
• rendere i contenuti facilmente referenziabili dall’AI

Struttura UX desiderata

Mantieni la sidebar principale della piattaforma come adesso, ma aggiungi una nuova voce:

• Documentazione

La pagina interna deve avere una struttura a 3 aree:

1. Colonna sinistra interna (navigazione documentazione)

Una sidebar secondaria con:

• barra di ricerca “Cerca nella documentazione”
• categorie principali
• pagine/guide annidate
• sezione attiva evidenziata
• stile coerente, non stile wiki old-school

Esempi categorie:

• Panoramica
• Tipi di violenza
• Difendersi
• Sicurezza digitale
• Raccolta prove
• Supporto e aiuto
• FAQ

2. Contenuto centrale

Area principale leggibile e ordinata con:

• breadcrumb in alto
• titolo della pagina docs
• piccolo sottotitolo/summary
• badge stato tipo “Pubblicata”
• contenuto articolato in sezioni con heading chiari
• paragrafi, liste, callout, box informativi, checklist
• un box finale tipo “Risorse collegate” o “Approfondisci”

Questa area deve sembrare una documentazione professionale, leggibile e rassicurante, non una dashboard vuota.

3. Colonna destra

Una colonna leggera con:

• “In questa pagina”
• anchor link alle sezioni
• un piccolo box “Chiedi all’AI su questo argomento”
• eventualmente un box “Ultimo aggiornamento”

Modalità da suggerire visivamente

La pagina deve far capire che in futuro potrà supportare ruoli diversi:

• utente → sola consultazione
• operatore → consultazione + modifica delle sezioni consentite
• admin → pieno controllo

Quindi inserisci nel design, in modo elegante e non invasivo:

• un pulsante Modifica in alto a destra
• eventualmente un badge ruolo o stato permessi
• ma senza trasformarla già in una schermata di editing completa

Stile dei contenuti

Usa contenuti d’esempio realistici sul tema:

• riconoscere segnali di violenza digitale
• cosa fare subito
• come raccogliere prove
• come proteggere i propri account
• a chi chiedere aiuto

Componenti desiderati

Inserisci nella pagina questi pattern UI:

• campo search docs
• navigation tree laterale
• article header
• callout box warning/info
• checklist box
• related resources card
• indice laterale destro
• CTA secondaria “Chiedi all’AI”

Vincoli importanti

• non cambiare il DNA visivo della piattaforma
• niente stile Notion puro
• niente stile GitBook generico
• niente docs troppo tecniche da developer tool
• deve sembrare una sezione naturale della piattaforma
• deve essere elegante, molto chiara e orientata alla consultazione
• mantenere il tono visivo già presente nei mockup e nella UI esistente

Output desiderato

Genera una schermata desktop completa della nuova pagina Documentazione della piattaforma, fedele allo stile attuale, pronta come base per una futura implementazione reale.
